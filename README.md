# LvsEngine

Early Lvs Engine version in modern C++ 23 with Qt6 UI and (temporary only limited to) Vulkan rendering.

The engine is required to run Studio and App (which is not available yet, even in the near future).

This will be expanding into online toy in the future!

## Project requirements

Windows 10+, and:

- CMake 3.27+
- Ninja
- MSYS2
- MinGW-w64 toolchain (`mingw64`)
- Qt6 (MSYS2 `mingw-w64-x86_64-qt6-base`, `mingw-w64-x86_64-qt6-tools`)
- Optional: Clang toolchain for ASAN (`clang64`)
- Vulkan SDK
  - `VULKAN_SDK` env var should point to your SDK install
  - `glslangValidator` or `glslc` must be available (used for shader compilation)

## Suggested MSYS2 packages

Run from MSYS2 shell:

```bash
pacman -S --needed mingw-w64-x86_64-toolchain mingw-w64-x86_64-cmake mingw-w64-x86_64-ninja mingw-w64-x86_64-qt6-base mingw-w64-x86_64-qt6-tools
```

Optional ASAN/Clang profile:

```bash
pacman -S --needed mingw-w64-clang-x86_64-toolchain mingw-w64-clang-x86_64-qt6-base mingw-w64-clang-x86_64-qt6-tools
```

## Build (CMake Presets)

Debug (MinGW default):

```powershell
cmake --preset studio-mingw-debug
cmake --build --preset studio-mingw-debug --parallel $env:NUMBER_OF_PROCESSORS
```

Release:

```powershell
cmake --preset studio-mingw-release
cmake --build --preset studio-mingw-release --parallel $env:NUMBER_OF_PROCESSORS
```

## Package / Bundle

Studio release bundle:

```powershell
cmake --build --preset studio-mingw-release --target lvs_studio_dist --parallel $env:NUMBER_OF_PROCESSORS
```

All app bundles:

```powershell
cmake --build --preset studio-mingw-release --target lvs_dist --parallel $env:NUMBER_OF_PROCESSORS
```

Bundles are written to:

```text
bundled/mingw-debug/...
bundled/mingw-release/...
```

## Runtime Dependency Vendoring (Project-local)

Packaging now caches runtime libraries into:

```text
third_party/runtime/<toolchain>-<config>/<target>/
```

This cache is generated by CMake during `*_dist` targets and reused by future package runs, so bundles no longer depend on broad DLL globbing from your local MSYS2 folders.

## VS Code

- Default configure/build preset is MinGW debug (`.vscode/settings.json`)
- Default build task is `studio-debug-mingw`
- MinGW debug/release launch configs are primary
- Clang ASAN tasks/config are kept for internal diagnostics
